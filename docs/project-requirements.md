# 项目需求 — Auto Content Platform

## 1. 软件目标

- **做什么**：自动化内容生成与多平台发布系统，面向产品评测/公司分析类 UGC 内容
- **解决什么问题**：批量生成高质量中文产品评测/公司分析文章，自动发布到知乎、微博等平台
- **目标用户**：内容运营人员，需要快速产出和分发内容
- **核心价值**：从内容生成到发布的完整自动化链路

## 2. 运行设备和平台

- Web 浏览器（管理控制台）
- Windows 本地开发（Redmi Book Pro 15 2022, AMD）
- 后端：Python FastAPI + uvicorn
- 前端：Vue 3 + Vite + Element Plus
- 数据库：PostgreSQL（主，192.168.0.170:5433），SQLite（降级）

## 3. 核心功能

| 优先级 | 功能 | 用户操作 | 预期结果 |
| --- | --- | --- | --- |
| 必须 | 内容生成（3 模式：纯产品/纯公司/混合） | 选模式 → 输入主题 → 生成 | 产出标题+正文+图片 |
| 必须 | 账号管理（多平台 cookie 导入/会话检测） | 添加账号 → 导入 cookie → 检测 | 账号状态实时反馈 |
| 必须 | 内容发布（知乎/微博） | 选内容+选账号 → 发布 | 文章成功发布到目标平台 |
| 应该 | 图片生成（Doubao-Seedream） | 自动触发或手动重新生成 | 文章配图生成 |
| 应该 | 发布状态追踪 | 查看发布记录 | 发布成功/失败/重试状态 |
| 可选 | 数据看板 | 查看统计 | 内容/发布数据可视化 |

## 4. 要解决的问题

- **痛点**：手动写文章 + 手动发布效率低
- **方案**：AI 生成内容 → 一键发布多平台
- **成功标准**：从输入主题到发布完成 ≤ 5 分钟

## 5. 展示内容

- 内容列表（标题、平台、状态、时间）
- 账号列表（平台、会话状态、登录检测）
- 发布记录（内容、账号、时间、状态）
- 内容编辑器（标题、正文预览）

## 6. 设计方向

- **风格**：深色工业风（Dark Industrial）
- **主色**：#0f1117（背景）+ #e8c547（强调暖金）
- **字体**：JetBrains Mono（代码/数据），系统默认（正文）
- **参考**：工具感 > 营销感

## 7. 交互和工作流

- 主流程：内容生成 → 预览 → 选择账号 → 发布
- 账号管理：添加 → 导入 cookie → 检测会话 → 就绪

## 8. 数据和集成

- 数据库：PostgreSQL（`ai_ugc@192.168.0.170:5433`）
- 外部 API：GPT API（内容生成）、Seedream API（图片）、Playwright（浏览器自动发布）
- 无官方 API 的功能直接取消

## 9. 验收标准

- [x] 三种内容生成模式可用
- [x] 知乎发布可用
- [ ] 微博发布可用（StorageState 过期问题待修复）
- [x] 账号管理（导入/检测）可用
- [x] 发布状态追踪可用
- [ ] 图片生成稳定（Seedream API 429 待处理）

## 10. 约束

- 截止时间：持续迭代
- 技术栈：Python FastAPI + Vue 3 + PostgreSQL
- 部署方式：本地开发运行
- 原则：无官方 API 支持的功能直接取消
